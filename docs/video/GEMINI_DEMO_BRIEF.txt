WHAT THIS PROJECT IS
--------------------------------------------------------------------------------
Personal automation / analytics workspace (Python + HTML + optional native
wrappers). Snapshot name: "Sweitzer Automations 3-22-26" (dated build label).

THREE MAIN STORYLINES FOR A DEMO VIDEO
1) Revenue Pulse — browser dashboard for sales CSVs (charts, KPIs). Engine:
   revenue_pulse/revenue_engine.py (Python standard library for CSV logic).
2) Flip profit tracker — browser UI for resale rows: estimates vs actuals,
   pending inventory, variance; engine: revenue_pulse/flip_engine.py.
3) CrewAI + Ollama — main.py with config/agents.yaml, config/tasks.yaml;
   tools can call sales_csv_summary / flip_csv_summary on sample CSVs. Local LLM.

PLATFORM WRAPPERS (same HTML dashboards, packaged)
- windows/ + windows_app/ — PyInstaller builds SweitzerAutomations-3-22-26.exe
- android/ — Gradle WebView app; copies revenue_pulse into assets at build
- apple/ — Xcode multiplatform WKWebView + custom URL scheme for local assets

COMMANDS THAT LOOK GOOD ON VIDEO
--------------------------------------------------------------------------------
# Dashboards (browser)
python3 -m http.server 8080 --directory revenue_pulse
# Then open http://localhost:8080 and http://localhost:8080/flip_tracker.html

# Engines only (terminal)
python3 -m revenue_pulse.revenue_engine revenue_pulse/sample_sales.csv
python3 -m revenue_pulse.flip_engine revenue_pulse/sample_flips.csv

# CrewAI (needs venv + Ollama + model)
source .venv/bin/activate
export PRODUCT="Sony WH-1000XM5"
export TARGET_PRICE="200"

# Tests (credibility)
pip install -r requirements-dev.txt
